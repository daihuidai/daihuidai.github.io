关于机具库限项目 Linux 环境部署方案二

基础工具：CentOS-7-x86_64-Minimal-1810.iso；VMware-workstation-full-14.0.0-6661328.exe

说明：以简短的语言讲述关于机具库限项目所使用的 Linux 在 VMware 下进行环境配置的过程

1. 原理

由于将在 Windows 下使用 VMware 来挂载 Linux 虚拟机从而运行项目，所以可以在本地通过在所提供对应版本的 VMware 下安装所提供的发行版本以及版本号的 Linux，预先完成项目所需要的环境配置，然后通过 VMware 克隆功能完整导出该独立的虚拟机，最后打包所有虚拟机文件用于后续部署时 VMware 使用。

2. 安装 Linux

使用所提供的 Linux 进行安装，由于使用的是最小化安装，所以服务器只有基础功能，部分需要功能需要进行添加。安装过程中需要在 网络与主机名 中勾选以太网连接，否则后续需要配置网络环境。如果没有勾选，则通过以下命令开启网络连接：

2.1. wget 安装

后面需要 wget ，通过 yum 命令可以方便的安装该工具（当然需要该系统中 /etc/yum.reops.d 有该信息，yum 软件管理包可以从指定服务器下载 RPM 包并安装）

2.2. 其他基础依赖

如有其它依赖需在此处添加

3. 配置环境

3.1. python 依赖

主要安装环境为 Python3 ，为保证后续源码编译安装 Python3 时不出现问题，提前配置其依赖。

3.2. 安装 python3.7.3 版本

为了减小服务器大小，这里不采用 Anaconda 来安装，直接编译 Python3 源码，配置环境变量后通过 pip 来安装其余 Python 包。

如果报错：提示没有合适的 C 编译器，则需要安装 yum -y install gcc 后执行 make clean 后再执行上下两条命令

3.2.1. 系统环境变量

如果可以，推荐配置系统环境变量，编辑 /etc/profile 文件，添加 python3 的 bin 路径。

可以键入 python3 检查是否正常。

3.2.2. 用户环境变量

如果权限有要求，则配置用户环境变量，编辑 /home/daihuidai/.bashrc 或 ~/.bashrc ，添加 Anaconda 的 bin 路径。

可以键入 python3 检查是否正常。

强调说明：服务器自带python为2.7，后续安装为python3.7.3，若要使用python3，需要指明python3、pip3等名称

3.3. 安装项目依赖包
3.3.1. Python3 环境

目前包括 pymysql、xgboost、lightgbm、pandas ，使用该顺序安装，pip 会同时安装包括 numpy、scipy、scikit-learn 等多个需要的依赖包。

3.3.2 其它环境

若有其它需求在此处补充。

4.1. Python3 环境测试

将挂载目录 usb 下的代码及数据集通过 cp 复制到用户目录下，使用 python3 来运行代码。

4.2. 其它环境测试

如有其它环境测试同样通过挂载 usb 来拷贝项目，通过相应工具来测试。

5. 克隆虚拟机

到此 Linux 的环境配置完成，接着需要将该虚拟机进行克隆，用于后续的部署，这点可以通过 VMware 自带克隆功能来完成。内容简单可以参考下面文章：

https://blog.csdn.net/pengpengpeng85/article/details/76382927

只需要部分参考即可，并且一定要选择 创建完整克隆 。保存好位置，后续转移可以使用 VMware 的打开虚拟机 功能来使用该克隆服务器。

6. 问题

过程中可能遇到的问题，上面其实在配置过程中已经解决了大部分的问题，如果后续还有问题，在此处分小结添加即可。

6.1. 转移克隆出现问题

当我将克隆文件在另一台电脑上挂载时，出现 intel vt-x 处于禁用状态 问题，很明确这是电脑没有开启该功能。

解决方法：以小新为例，通过 fn+F2 进入 BIOS ， 移动到 Configuration 下，下移到 Intel Virtual Technology 处使用 ENTER 改变 Disabled 为 Enabled，再按 F10 保存退出。

说明：电脑不同路径会有差异，但最终需要的只是将 Intel Virtual Technology 改为 Enabled

6.2. Python 代码执行报错

将 Windows 下写好的 Python 代码迁移到 Linux 下，通过 python3 test.py 可以正常执行。如果想像脚本一样直接运行，需要在代码第一行加入 python 的解释器位置：

添加了解释器就可以直接 ./test.py 运行代码了，但是还是报错了： No such file or directory 。这是因为文件格式兼容性问题。

解决方法：通过 vi 进入 py 文件，键入 esc 后ctrl+: ，进入命令行模式，输入 :set ff ,可以看到显示为 fileformat=dos , 所以命令行重新设置文件格式： :set ff=unix ， 最后 :wq 保存退出再运行即可。
